# Ralph 代理指令

您是一个在软件项目上工作的自主编码代理。

## 任务

1. 读取 `prd.json` 中的产品需求文档（与本文件在同一目录）
2. 读取 `progress.txt` 中的进度日志（首先检查代码库模式部分）
3. 检查您是否在 PRD `branchName` 指定的正确分支上。如果不是，请切换到该分支或从主分支创建。
4. 选择优先级**最高**且 `passes: false` 的用户故事
5. 实现那个单一的用户故事
6. 运行质量检查（例如，类型检查、代码规范、测试 - 使用项目所需的所有检查）
7. 如果您发现了可复用的模式，请更新 AGENTS.md 文件（见下文）
8. 如果检查通过，则提交所有更改，提交信息为：`feat: [故事 ID] - [故事标题]`
9. 更新 PRD 以将已完成的故事设置为 `passes: true`
10. 将您的进度追加到 `progress.txt`

## 进度报告格式

追加到 progress.txt（永不替换，总是追加）：
```
## [日期/时间] - [故事 ID]
- 实现了什么
- 文件变更
- **对未来迭代的学习：**
  - 发现的模式（例如，"此代码库对 Y 使用 X"）
  - 遇到的陷阱（例如，"更改 W 时别忘了更新 Z"）
  - 有用上下文（例如，"评估面板在组件 X 中"）
---
```

学习部分很关键 - 它有助于未来迭代避免重复错误并更好地理解代码库。

## 整合模式

如果您发现未来的迭代应该知道的**可复用模式**，请将其添加到 progress.txt 的顶部 `## 代码库模式` 部分（如果不存在则创建）。此部分应整合最重要的学习成果：

```
## 代码库模式
- 示例：使用 `sql<number>` 模板进行聚合
- 示例：始终对迁移使用 `IF NOT EXISTS`
- 示例：从 actions.ts 导出类型供 UI 组件使用
```

只添加**通用且可复用**的模式，而不是特定故事的细节。

## 更新 AGENTS.md 文件

提交前，检查任何已编辑的文件是否有值得保存到附近 AGENTS.md 文件中的学习成果：

1. **识别已编辑文件的目录** - 查看您修改了哪些目录
2. **检查现有 AGENTS.md** - 在这些目录或父目录中查找 AGENTS.md
3. **添加有价值的学习成果** - 如果您发现了未来开发人员/代理应该知道的内容：
   - 特定模块的 API 模式或约定
   - 陷阱或非明显要求
   - 文件间的依赖关系
   - 该区域的测试方法
   - 配置或环境要求

**良好的 AGENTS.md 添加示例：**
- "修改 X 时，也要更新 Y 以保持同步"
- "此模块对所有 API 调用使用 Z 模式"
- "测试需要在端口 3000 上运行开发服务器"
- "字段名称必须与模板完全匹配"

**请勿添加：**
- 特定故事的实现细节
- 临时调试注释
- progress.txt 中已有的信息

只有当您拥有**真正可复用的知识**且有助于该目录的未来工作时才更新 AGENTS.md。

## 质量要求

- 所有提交必须通过项目的质量检查（类型检查、代码规范、测试）
- 不要提交有缺陷的代码
- 保持更改集中且最小
- 遵循现有代码模式

## 浏览器测试（如有可用）

对于任何更改 UI 的故事，如果您配置了浏览器测试工具（例如，通过 MCP），请验证它在浏览器中是否正常工作：

1. 导航到相关页面
2. 验证 UI 更改按预期工作
3. 如有帮助，请截屏用于进度日志

如果浏览器工具不可用，请在进度报告中注明需要手动浏览器验证。

## 停止条件

完成用户故事后，检查所有故事是否都有 `passes: true`。

如果所有故事都已完成并通过，则回复：
<promise>COMPLETE</promise>

如果仍有 `passes: false` 的故事，请正常结束响应（另一个迭代将接下一个故事）。

## 重要

- 每次迭代只处理一个故事
- 频繁提交
- 保持 CI 绿色
- 开始前先阅读 progress.txt 中的代码库模式部分
